# 產品簡易抽驗紀錄表清理與診斷工具

這是可在本機或 GAI Canvas 測試的 React + Vite App。

## 這版新增功能

除了原本的清理後明細資料、KPI、異常資料清單與下載功能外，已加入：

1. **週別統計 KPI（可篩選）**
   - 可依月份、週別、廠商篩選。
   - 顯示明細筆數、廠商數、有效日期數、整體合格率、重量/長度/寬度比率、其他問題率。

2. **月統計摘要**
   - 依月份彙總明細筆數、週別數、廠商數、有效日期數、整體合格率與各規格比率。

3. **老闆彙總檔格式**
   - 依目前篩選的月份、週別、廠商產出主管閱讀表。
   - 顯示該週期該廠商的產品有效日期。
   - 依規格計算 Result 百分比：
     - 重量：60~70g、<60g、60~62g、68~70g、>70g
     - 長度：12~16cm、12~13cm、>16cm、<12cm
     - 寬度：3.5~4.5cm、3.5~3.6cm、<3.5cm、>4.5cm
     - 其他問題：其他問題欄非空白占比
   - UI 會以反色顯示低於或高於標準規格。

4. **下載 Excel 增加工作表**
   - 老闆彙總表_目前篩選
   - 週別統計_目前篩選
   - 月統計摘要
   - 彙總檔格式_全部
   - 清理後明細資料
   - 報表問題診斷表
   - KPI摘要
   - 異常資料清單
   - 排除工作表清單
   - 處理規則說明

## 真實 Excel 解析邏輯

本工具不是假資料展示，會使用：

- `FileReader` 讀取使用者上傳的 Excel
- `XLSX.read(arrayBuffer, { type: "array" })` 解析活頁簿
- `workbook.SheetNames` 讀取所有工作表
- `XLSX.utils.sheet_to_json(worksheet, { header: 1, defval: "" })` 轉成二維陣列
- 依第 7 列偵測「每片重量」欄位，將橫向廠商區塊轉為直向資料
- `XLSX.writeFile` 搭配 Blob 備援下載 Excel 結果檔

## 注意事項

- `xlsx` 社群版對 Excel 儲存格樣式支援有限，因此匯出的 Excel 以「比較結果」欄位保留高於/低於標準規格；App 畫面本身會用顏色反白提示。
- 程式判定結果是初步檢查，正式稽核前仍需要人工檢核。

## 本版修正：廠商篩選

- 修正「廠商名稱」被誤判成「名稱」的問題。
- 廠商擷取時會優先讀取「廠商名稱」右側真正的廠商值。
- 週別統計區的廠商下拉選單會依目前選定的月份與週別動態列出可選廠商。
- 切換月份會清空週別與廠商，切換週別會清空廠商，避免舊篩選條件殘留。

## 本版修正：廠商括號子公司統計

- 廠商名稱若為 `母公司(子公司)` 或 `母公司（子公司）`，清理後明細會保留完整原始廠商名稱。
- 新增欄位：`統計廠商`、`子公司`。
- 週別統計、月統計、老闆彙總表與廠商篩選會使用 `統計廠商` 分組。
- 範例：`泰安(泰立富)` 會被歸入統計廠商 `泰安`，括號內 `泰立富` 不會被排除，會一併計入該公司的百分比與樣本筆數。
- 老闆彙總表會顯示「原始廠商/子公司」，方便人工確認哪些子公司已被併入統計。

## 本版更新：拖拉上傳

上傳區支援兩種方式：

1. 點選「選擇 Excel 檔案」上傳 `.xlsx`、`.xls`、`.xlsm`。
2. 直接把 Excel 檔案拖拉到上傳框，放開後自動讀取並解析。

拖拉上傳仍使用同一套真實解析流程：`FileReader`、`XLSX.read`、`XLSX.utils.sheet_to_json({ header: 1 })`。

## 本版更新：連續小數點自動修正

- 若數值欄位出現 `67..2`、`67...2` 這類常見人工 keyin 手誤，系統會自動轉成 `67.2` 後納入統計。
- 修正後的資料列不會被當成無法判定；但會在「資料品質備註」標示「疑似多輸入小數點，已自動修正，請人工確認」。
- 若格式不明確，例如 `6.7.2`，仍會列為數值格式異常，不會自行猜測。

## 本版修正：小數點自動修正後廠商篩選恢復

- 保留 `67..2`、`67...2` 自動修正為 `67.2` 的功能。
- 保留拖拉上傳、母公司/子公司合併統計、週別統計、月統計與老闆彙總表。
- 修正部分檔案在加入自動修正後，統計廠商下拉選單只剩「全部統計廠商」的問題。
- 解析時會補強合併儲存格讀取，避免合併儲存格導致廠商、效期或抽驗者被讀成空白。
- 若明細工作表廠商欄位因格式偏移抓不到，會嘗試參考同一活頁簿中的「彙整 / 報告」工作表，以相同月份、週別與區塊順序補入廠商名稱，並在資料品質備註標示需人工確認。

## 本版更新：資料匣 / 多檔批次上傳

- 上傳區新增「選擇資料匣」按鈕，可一次選取整個資料匣內的 Excel 檔案。
- 支援多檔 Excel 上傳，包含 `.xlsx`、`.xls`、`.xlsm`。
- 拖拉上傳支援拖拉單一 Excel、多个 Excel，若瀏覽器支援，也可直接拖拉整個資料匣。
- 資料匣內非 Excel 檔案會自動略過，並列入「排除工作表清單」供人工檢查。
- 多個 Excel 會合併成同一份清理後明細資料、週別統計、月統計與老闆彙總表。
- 清理後「來源工作表」會加上檔案路徑，方便區分不同 Excel 檔案中同名工作表。

注意：資料匣上傳使用瀏覽器提供的 `webkitdirectory` 與拖拉目錄 API；Chrome、Edge 支援度較佳，部分瀏覽器可能只支援多檔上傳，不支援整個資料匣拖拉。
